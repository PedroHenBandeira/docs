## Como utilizar este endpoint?

A ativação de aplicativo é feita por requisições **POST**.  
Escolha o caminho de acordo com o tipo de app que deseja ativar.

<ParamField
    path="/lines/active/app/xstream"
    type="string"
    placeholder="POST /lines/active/app/xstream"
    required
>
    Endpoint para ativar apps do tipo **XStream** (ex.: WTV Player / Wapp).
</ParamField>

<ParamField
    path="/lines/active/app"
    type="string"
    placeholder="POST /lines/active/app"
    required
>
    Endpoint genérico para ativar apps como **BrasilIPTV**.
</ParamField>

### Headers obrigatórios

<ParamField
    header="Authorization"
    type="string"
    placeholder="Bearer {token}"
    required
>
    Token de autenticação no formato **Bearer**.
</ParamField>

<ParamField
    header="Content-Type"
    type="string"
    placeholder="application/json"
    required
>
    Deve ser sempre **application/json**.
</ParamField>

### Parâmetros do corpo (payload)

- **nameApp**: string — nome do aplicativo de reprodução
- **namePlaylist**: string — nome da playlist ou pacote
- **mac**: string — endereço MAC do dispositivo (com ou sem “:”)
- **id_user**: number — ID do usuário final

### Exemplo de Requisição — XStream

````http
POST https://mcapi.knewcms.com:2087/lines/active/app/xstream
Authorization: Bearer seu_token_aqui
Content-Type: application/json

{
  "nameApp": "WTV Player/Wapp",
  "namePlaylist": "teste 1",
  "mac": "234235333",
  "id_user": 102330893
}

### Resposta

```json
  status="success"
  type="object"

{ "success": true }

```json
{"message":"Dispositivo não encontrado","error":"Not Found","statusCode":404}
````
