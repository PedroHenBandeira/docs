## Como utilizar este endpoint?

Esse endpoint ativa **vários aplicativos** na plataforma.

### Headers obrigatórios

<ParamField
  header="Authorization"
  type="string"
  placeholder="Bearer {token}"
  required
>
  O token de autenticação no formato Bearer é obrigatório para acessar este
  endpoint.
</ParamField>

### Parâmetros do corpo da requisição (Payload)

<ParamField body="notes" type="string" placeholder="Observações">
  Observações adicionais sobre o cliente. Campo opcional.
</ParamField>

### Exemplo de Requisição

```json
POST https://mcapi.knewcms.com:2087/lines/test
Headers:
{
  "Authorization": "Bearer seu_token_aqui",
  "Content-Type": "application/json"
}
Payload:
{
    "isTrial": 0,
    "username": "xxxxxxxx",
    "password": "xxxxxxxx",
    "whatsapp": "xxxxx",
    "telegram": "xxxx",
    "country": "Brasil",
    "notes": "xxxx",
    "credits": 1,
    "email": "xxxx"
}
```

### Resposta

<ResponseField name="status" type="number">
  Indica o código HTTP da resposta. Esperado: 201 (activated).
</ResponseField>

<ResponseField name="code" type="string">
  O código de status da resposta.
</ResponseField>

<ResponseField name="response" type="object">
  Conteúdo da resposta.
</ResponseField>

### Exemplo de Resposta - Sucesso

<ResponseExample>
```json
{
  "success": true
}
````
<ResponseExample/>
### Exemplo de Resposta - Error

<ResponseExample>
```json
{"message":"Dispositivo não encontrado","error":"Not Found","statusCode":404}
````
<ResponseExample/>
